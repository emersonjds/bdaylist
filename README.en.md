## ✨ About the product

**BdayList** is a **birthday gift registry** platform. The birthday person (host) creates the event, builds the list and shares a high-entropy link; guests open it, RSVP, and **reserve** the gift they'll bring.

> 💡 **Important:** there is **no money flowing through the app** yet. "Gift" = **reserve**. The present is arranged and delivered off-platform. Goals / "% raised" are a future direction (likely external PIX), out of the current scope.

### Who does what

| 🎂 Host (birthday person)                             | 🎁 Guest                          |
| ----------------------------------------------------- | --------------------------------- |
| Creates event + list and shares the link              | Opens it via the shareable link   |
| Adds/edits gifts (image, reference price, store link) | RSVPs                             |
| Flags "Most Wanted" and "Group Gift"                  | Reserves the gift (no duplicates) |
| Tracks confirmed guests and activity                  | Leaves a note on the wall         |
| **"Surprise"** option: not seeing who reserved        | Filters by price and searches     |

## 📐 Architecture — Feature-Sliced Design

```
src/
├── app/        ← Routes and layouts (App Router). No business logic.
├── widgets/    ← Composed UI (header, gift-grid, dashboard, landing…)
├── features/   ← Use cases (reserve-gift, rsvp, manage-gifts, auth…)
├── entities/   ← Domain models (event, gift, reservation, guest…)
└── shared/     ← Reusable infra (ui, lib, providers)
```

**Import rule:** only from **layers below** — `app → widgets → features → entities → shared`. Never the other way, never sideways between slices of the same layer.

## 🚀 Getting started

**Prerequisites:** Node.js 20+ and pnpm 10.

```bash
# 1. Install dependencies
pnpm install

# 2. Set up the environment (MSW is on while Supabase doesn't exist yet)
cp .env.local.example .env.local

# 3. Run in development
pnpm dev          # http://localhost:3000

# 4. Production static build
pnpm build        # generates ./out
```

`.env.local`:

```ini
NEXT_PUBLIC_API_MOCKING=enabled        # turns MSW on in the browser
NEXT_PUBLIC_SUPABASE_URL=              # future
NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY=  # future (public key; protection is RLS)
```

## 🧪 Quality & tests

Every feature carries **three layers** of tests — and E2E counts more than the mocks:

1. **Unit** — pure logic (libs, derivations, rules).
2. **Integration (MSW)** — fetchers/queries against the mocked backend.
3. **E2E (Playwright)** — real browser flow, mobile (375px) and desktop (1280px), with evidence screenshots.

```bash
pnpm test            # Vitest in watch mode
pnpm test:run        # Vitest once
pnpm test:coverage   # coverage (v8)
pnpm test:e2e        # Playwright (mobile + desktop)
pnpm validate        # type-check + lint + format:check + tests
```
